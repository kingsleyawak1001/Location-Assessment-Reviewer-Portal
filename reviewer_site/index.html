Pipeline Stages (Clear Stage-by-Stage View)

Stage 1: Ingestion and Quality

  • Normalize timestamp/schema.
  • Validate coordinates and duplicates.
  • Split accepted/rejected with reason tracking.

Stage 2: Transformation

  • Segment by device, time, and distance.
  • Apply accuracy-aware and night-gap rules.
  • Classify stay/pass_by and stay_type.

Stage 3: Persistence and Lineage

  • Persist visits into query-ready store.
  • Attach run lineage and artifact references.
  • Materialize run-level aggregates.

Stage 4: Serving and Validation

  • Expose map and journey APIs.
  • Support CLI query/export commands.
  • Run lint/type/tests/e2e validation policy.
